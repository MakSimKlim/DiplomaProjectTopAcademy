1. Должна быть микросервисная архитектура
2. Базы данных для каждого заказчика свои (Авторизация + Объекты строительства)
3. Характеристики материалов из ГОСТ - некие табличные константы, хранящиеся по-умолчанию в каждой DB
4. Смысл каждой DB в том для каждого заказчика: на каждый объект строительства информация вносится один раз и этой информацией пользуются все микросервисы. DB становится неким ядром, из которого берут данные следующие приложения: для подсчета объемов, составления заявок, ведения исполнительной документации, составления ведомостей объемов работ для сметного расчета, расчета заработной платы, составления графиков производства работ, составления ежемесячных журналов выполнения работ, статистических дашбордов, аналитические сервисы и т.д.
